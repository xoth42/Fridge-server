# Changelog

## Phase A — Foundation (2026-03-18)

### New files
- `.env.example` — complete configuration template with comments for every variable
- `config/caddy/Caddyfile` — Caddy reverse proxy, auto-TLS via Let's Encrypt, iframe embedding support
- `config/ddclient/ddclient.conf.template` — ddclient template for name.com DynDNS2
- `config/alertmanager/alertmanager.yml.template` — full Slack + email template, generated by install.sh via envsubst
- `planning/phase-a-notes.md` — architecture decisions, service list, open issues
- `planning/changelog.md` — this file

### Modified files
- `docker-compose.yml`
  - Added: Caddy (reverse proxy, `production` profile), ddclient (DynDNS, `production` profile), Watchtower (monitor-only)
  - Added: `grafana-data` named volume so Grafana SQLite DB (alert rules, user edits) persists across restarts
  - Added: `alertmanager-data` named volume
  - Added: `caddy_data`, `caddy_config` volumes for TLS cert persistence
  - Prometheus: added `--storage.tsdb.retention.time` and `--storage.tsdb.retention.size` from env vars
  - Grafana: added anonymous access env vars, allow-embedding, SMTP config, admin creds from env
  - All services: `restart: unless-stopped`
  - Compose profiles: `production` profile for caddy + ddclient (not started in CI)

- `install.sh` — full rewrite
  - Idempotent (safe to re-run)
  - Coloured output, step headers
  - Preflight: Docker, Compose plugin, envsubst
  - Validates .env (required vars, stricter in production than CI)
  - Generates `alertmanager.yml` from template via envsubst
  - Generates `ddclient.conf` from template (production only)
  - ufw firewall rule for port 9091 (if ALLOWED_PUSH_CIDR is set)
  - Starts stack with `--profile production` in production, without in CI
  - Health check polling for all 4 core services
  - Creates Grafana lab user via API (production only)
  - Prints summary with URLs and Pushgateway address

- `config/grafana/provisioning/dashboards/dashboard-provider.yml`
  - `disableDeletion: false` (was true)
  - `allowUiUpdates: true` (was false)
  - User dashboard edits now persist to the grafana-data volume

- `.github/workflows/ci.yml`
  - Installs `gettext` (provides envsubst)
  - Creates a minimal `.env` with test credentials before running install.sh
  - Calls `./install.sh` (CI=true auto-detected from GitHub Actions env)
  - Uses correct admin password from .env for Grafana API checks (was hardcoded admin:admin)

- `.gitignore`
  - Added `config/ddclient/ddclient.conf` (generated, never commit)
  - Documented that `config/alertmanager/alertmanager.yml` is tracked in git but modified by install.sh — do not commit after running install.sh

### Fridge-data-uploader changes
- `example_data/collectall.py`: fixed path bug — was looking for `.env` in `example_data/` instead of the repo root
